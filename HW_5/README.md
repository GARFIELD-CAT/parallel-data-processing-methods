# Практическое задание 5: Потокобезопасные коллекции

## Краткое описание

Проект демонстрирует использование потокобезопасных коллекций `ConcurrentDictionary`, `BlockingCollection`, `ConcurrentBag` в .NET для построения конкурентных систем без ручной синхронизации (`lock`, `Monitor` и т.д.). Реализованы:

- Потокобезопасный каталог книг (`ConcurrentLibraryCatalog`)
- Очередь задач с ограниченной ёмкостью (`TaskQueueManager`)
- Кэш с автоматическим истечением элементов (`ConcurrentCache`)
- Сравнительные тесты производительности между `ConcurrentDictionary` и обычным `Dictionary` с `lock`

## Требования

- .NET SDK 10.0 (используется C# с поддержкой .NET 10)
- Среда разработки: Visual Studio 2022+, Rider, VS Code или любая другая

## Как собрать и запустить

1. Клонируйте/скопируйте файлы проекта в папку (например, `Task5_ConcurrentCollections`).
2. Убедитесь, что в папке есть файл `.csproj` (если нет, создайте новый проект: `dotnet new console` и замените файлы).
3. Выполните в терминале:
   ```bash
   dotnet restore
   dotnet run
   ```
