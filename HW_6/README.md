# Практическое задание 6: Атомарные операции и Interlocked

## Описание проекта

Данный проект демонстрирует использование атомарных операций через класс `Interlocked` в .NET 10 для построения потокобезопасных структур данных без применения блокировок (`lock`, `Monitor` и т.п.). Реализованы:

- Атомарный счётчик `AtomicCounter`;
- Lock-free стек `LockFreeStack<T>`;
- Трекер статистики `StatisticsTracker`;
- Бенчмарк для сравнения производительности атомарных операций и традиционных блокировок.

Цель — показать преимущества lock-free подхода при высокой конкурентной нагрузке.

## Требования

- .NET SDK 10.0.201 или совместимая версия (язык C#).
- Среда разработки, поддерживающая .NET (Visual Studio, Rider, VS Code) или командная строка.

## Структура классов

### `AtomicCounter`

Потокобезопасный счётчик на основе `Interlocked.Increment`, `Decrement`, `Add`, `Exchange`, `CompareExchange`. Все операции чтения и изменения атомарны. Не использует блокировки.

### `LockFreeStack<T>`

Реализация стека на односвязном списке с применением `Interlocked.CompareExchange` (CAS). Методы `Push`, `TryPop`, `TryPeek`, `IsEmpty`, `Clear` работают без блокировок и гарантируют корректность при конкурентном доступе.

### `StatisticsTracker`

Сборщик статистики, использующий `Interlocked.Increment` и `Interlocked.Add` для учёта количества и времени запросов. Безопасен для многопоточного использования без блокировок.

### `AtomicBenchmark`

Статический класс с методами для измерения производительности:

- `BenchmarkInterlockedCounter` – тест `AtomicCounter` с заданным числом операций и потоков;
- `BenchmarkLockedCounter` – тест обычного счётчика с `lock`;
- `BenchmarkLockFreeStack` – тест lock-free стека;
- `BenchmarkStatisticsTracker` – тест трекера статистики;
- `CompareAllApproaches` – комплексное сравнение и вывод результатов.

### `Program.cs`

Точка входа. Задаёт параметры тестов (количество операций, потоки, фиксированный seed `42`), запускает все тесты, проверяет корректность (для счётчика — итоговое значение 0, для стека — соответствие количества элементов ожидаемому) и выводит сводную статистику.

## Пошаговое объяснение логики

1. **Атомарные операции (Interlocked)**  
   Методы `Interlocked.Increment`, `Decrement`, `Add` гарантируют, что операции чтения-изменения-записи выполняются как единое неделимое действие на уровне процессора. Это исключает гонки данных без использования блокировок.

2. **CompareExchange (CAS)**  
   `Interlocked.CompareExchange(ref location, newValue, comparand)` атомарно сравнивает `location` с `comparand` и, если они равны, заменяет на `newValue`. Возвращает исходное значение. Используется в lock-free стеке для обновления головы.

3. **Lock-free стек**  
   Алгоритм Push:

- Создаём новый узел.
- Запоминаем текущую голову.
- Устанавливаем `Next` нового узла на эту голову.
- Пытаемся сделать новый узел головой через CAS. Если за время подготовки голова изменилась, CAS вернёт другое значение, и мы повторяем.
  Алгоритм Pop аналогичен, но сдвигает голову на следующий узел.

4. **Сравнение производительности**  
   В тестах используется `Parallel.For` для одновременного выполнения операций из множества потоков. Измеряется время выполнения. Сравниваются:

- Атомарный счётчик и счётчик с `lock`: видно, что Interlocked быстрее, так как избегает накладных расходов на блокировки и переключения контекста.
- Lock-free стек: демонстрирует корректную работу при 50 потоках и 100 000 операций.
- StatisticsTracker: обрабатывает 50 000 запросов в 200 потоков, используя только атомарные инкременты.

5. **Проверка корректности**

- Счётчик: после равного числа инкрементов и декрементов значение должно быть 0.
- Стек: подсчитывается количество оставшихся элементов и сравнивается с разницей push и успешных pop. Совпадение говорит о целостности данных.
- Трекер: корректность гарантируется атомарностью операций.

## Заключение

Атомарные операции на основе `Interlocked` позволяют создавать быстрые и масштабируемые потокобезопасные компоненты без блокировок. Они особенно эффективны при очень высокой частоте обращений, где обычные блокировки становятся узким местом. Проект наглядно демонстрирует это преимущество.
