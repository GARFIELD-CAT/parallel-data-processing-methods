# Практическое задание 7: Основы TPL (Task Parallel Library)

## Краткое описание

Проект демонстрирует использование Task Parallel Library в C# для параллельной обработки данных, построения цепочек задач с продолжениями, обработки исключений и отмены операций, а также сравнения производительности TPL с ручным управлением потоками (Thread).

## Требования

- .NET SDK 10.0.201 или совместимый.
- Собрать проект: `dotnet build`
- Запустить: `dotnet run`

## Структура классов и их назначение

| Файл                       | Назначение                                                                                                                          |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `Program.cs`               | Точка входа, создание тестовых данных, запуск всех тестов, вывод сводной статистики.                                                |
| `TaskDataProcessor.cs`     | Обработка массивов с помощью `Task.Run`, `Task.Factory.StartNew`, с поддержкой отмены (`CancellationToken`) и таймаутов.            |
| `TaskChainBuilder.cs`      | Построение цепочек задач через `ContinueWith`, включая обработку успеха, ошибок, отмены, ветвление (`WhenAny`) и повторные попытки. |
| `ExceptionHandlingDemo.cs` | Демонстрация перехвата и обработки `AggregateException` разными способами (Wait, WhenAll, ContinueWith).                            |
| `TPLBenchmark.cs`          | Измерение производительности Task.Run, Task.Factory.StartNew и Thread, сравнение и вывод результатов.                               |
| `README.md`                | Этот файл с описанием проекта.                                                                                                      |

## Пошаговое объяснение логики

### 1. TaskDataProcessor

- `ProcessDataAsync` разбивает массив на 4 равные части и обрабатывает каждую через `Task.Run`. Ожидание всех задач через `Task.WhenAll`, объединение результатов в один массив.
- `ProcessDataInParallel` делает то же, но с возможностью задать степень параллелизма (количество задач).
- `ProcessDataWithFactory` аналогичен, но использует `Task.Factory.StartNew` с явным указанием планировщика `TaskScheduler.Default`.
- `ProcessDataWithCancellation` при каждой итерации цикла вызывает `cancellationToken.ThrowIfCancellationRequested()`, что позволяет прервать обработку при запросе отмены.
- `ProcessDataWithTimeout` создаёт `CancellationTokenSource` с заданным временем и передаёт его в метод с отменой. После завершения задачи токен корректно освобождается.

### 2. TaskChainBuilder

- `BuildProcessingChain`: первая задача копирует данные, затем `ContinueWith` с `OnlyOnRanToCompletion` увеличивает элементы на 10%. Дополнительные продолжения с `OnlyOnFaulted` и `OnlyOnCanceled` только логируют ситуацию. Последнее звено (с `ExecuteSynchronously`) выводит сумму.
- `BuildComplexChain`: запускает две параллельные задачи (быструю и медленную), `Task.WhenAny` выбирает первую завершившуюся и применяет к её результату финальную обработку.
- `BuildParallelChain`: для каждого набора данных из переданного массива запускает отдельную цепочку `BuildProcessingChain` и ожидает все через `Task.WhenAll`.
- `BuildRetryChain`: создаёт задачу, которая может выбросить исключение (деление на 0). Если задача упала, `ContinueWith` запускает новую попытку до `maxRetries` раз.

### 3. ExceptionHandlingDemo

- `HandleExceptionsWithWait`: запускает хорошую и плохую задачу, ждёт через `Task.WaitAll`, ловит `AggregateException` и передаёт в `HandleAggregatedExceptions`.
- `HandleExceptionsWithWhenAll`: аналогично, но ожидание идёт через `Task.WhenAll().Wait()`.
- `HandleExceptionsWithContinueWith`: к задаче, бросающей исключение, прикрепляется продолжение `OnlyOnFaulted`, которое выводит сообщение и предотвращает необработанное исключение.
- `HandleAggregatedExceptions`: использует `Flatten()` для упрощения вложенных `AggregateException` и `Handle()` для избирательной обработки известных типов исключений.

### 4. TPLBenchmark

- Во всех трёх методах (`BenchmarkTaskRun`, `BenchmarkTaskFactory`, `BenchmarkThreadAPI`) выполняется `operationCount` атомарных инкрементов (`Interlocked.Increment`), разделённых между `threadCount` рабочими единицами (задачи или потоки). Измеряется время и подсчитываются успешные операции.
- `CompareAllApproaches` вызывает все три бенчмарка с фиксированными параметрами (100 000 операций, 4 потока) и выводит результаты в требуемом формате, включая ускорение TPL относительно Thread API.

### 5. Program.cs

- Генерирует 10 000 000 случайных чисел с `Random(42)`.
- Создаёт несколько наборов данных поменьше.
- Последовательно тестирует все компоненты: обработку данных с разной степенью параллелизма, отмену и таймауты, цепочки задач, обработку исключений и сравнение производительности.
- Проверяет корректность путём сравнения обработанных значений с ожидаемой формулой.
- Выводит итоговую сводку.
