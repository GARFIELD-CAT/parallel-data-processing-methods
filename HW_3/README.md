# Практическая работа 3: Синхронизация с помощью lock и Monitor в .NET

## Цель работы

Освоить механизмы синхронизации потоков с использованием `lock` и `Monitor`, понять причины возникновения гонок данных и взаимных блокировок (deadlock), а также научиться измерять накладные расходы на синхронизацию.

## Краткое описание реализации

Программа моделирует систему обработки банковских транзакций в многопоточной среде. Реализованы три способа доступа к общему ресурсу (счёту):

1. **Без синхронизации** – операции `Deposit`, `Withdraw`, `Transfer` изменяют баланс напрямую. При одновременном выполнении несколькими потоками возникают гонки данных, итоговый баланс искажается.
2. **С использованием оператора lock** – блок кода защищается объектом блокировки, гарантируя, что в каждый момент времени его выполняет не более одного потока.
3. **С использованием класса Monitor** – явные вызовы `Monitor.Enter` и `Monitor.Exit` с обязательным освобождением в блоке `finally`. Дополнительно реализованы методы с таймаутами (`TryEnter`), возвращающие успешность захвата блокировки.

Также демонстрируется проблема **deadlock** при параллельных переводах между счетами и её решение с помощью **единообразного порядка захвата** блокировок (упорядочивание по хэш-кодам объектов блокировки).

## Структура проекта

- **Program.cs** – точка входа, создание тестовых данных, запуск тестов и вывод результатов.
- **BankAccount.cs** – класс банковского счета с тремя группами методов (без синхронизации, lock, Monitor).
- **TransactionProcessor.cs** – запуск транзакций в отдельных потоках (`Thread`) и демонстрация deadlock'а.
- **SynchronizationBenchmark.cs** – замеры времени выполнения и корректности для каждого подхода.
- **README.md** – текущий файл.

## Как запустить

1. Убедитесь, что установлен .NET SDK (в нашем случае 10.0.201).
2. Поместите все файлы `.cs` и `README.md` в одну папку (или создайте консольный проект и замените файлы).
3. Выполните в терминале:
   ```bash
   dotnet run
   ```
